# Презентација: Прашања и Одговори (Cheat Sheet)

## 0. Стартување на Апликацијата (Start-up)
Ако го вклучуваш лаптопот од почеток:
1.  Отвори **VS Code**.
2.  Оди во **File > Open Folder** и избери ја папката `RevijaRestaurantOrderingSystem` (на Desktop).
3.  Отвори нов Терминал: **Terminal > New Terminal**.
4.  Во терминалот напиши ја командата:
    ```bash
    dotnet run --project Web
    ```
5.  Почекај да се испише текстот `Now listening on: http://localhost:5174`.
6.  Во прелистувач (Chrome/Edge) отвори ја адресата:
    *   [http://localhost:5174/swagger](http://localhost:5174/swagger)
    *   (Ако не се отвори автоматски, копирај го линкот или држи `Ctrl` + клик на линкот во терминалот и додај `/swagger` на крајот).

---

Ако професорите сакаат да видат "акција", еве како да се справиш со нивните барања.

## 1. "Ајде направи една нарачка да видиме како работи"
**Одговор**: "Секако, ќе креирам нарачка за клиентот 'Marko' (ID: 1)."

**Чекори во Swagger**:
1.  Отвори **POST /api/orders**.
2.  Кликни **Try it out**.
3.  Во прозорецот залепи го овој JSON (копирај го од тука):

```json
{
  "customerId": 1,
  "items": [
    {
      "menuItemId": 1,
      "quantity": 2
    },
    {
      "menuItemId": 2,
      "quantity": 1
    }
  ]
}
```
4.  Кликни **Execute**.
5.  Покажи им го резултатот доле: ќе врати `totalPrice` пресметано автоматски и статус "Order Placed".

---

## 2. "Ајде додади нов клиент или ново јадење"
**Внимавај**: Ние *немаме* API (копче) за додавање клиенти/јадења, туку само за читање (GET).

**Одговор**:
"Бидејќи ова е Customer App, менаџирањето на менито и клиентите е предвидено за Admin панел кој не е дел од ова API. Но, можам веднаш да ги додадам преку **Seed** кодот."

**Ако инсистираат да додадеш:**
1.  Отвори го фајлот `Repository/DbSeeder.cs` во VS Code.
2.  Во листата `burgers` (или `drinks`), додади нов ред:
    ```csharp
    new MenuItem { Name = "Professors Burger", Description = "Special edition", Price = 300m, Category = "Burgers", IsAvailable = true }
    ```
3.  Зачувај (`Ctrl+S`).
4.  Во терминал: `git add .`, `git commit -m "Add special burger"`, `git push`.
5.  Кажи им: "Сега се прави deployment, и за 2 минути ќе се појави на Swagger."

---

## 3. "Како да додадам нов град за Времето?"
**Одговор**: "Нема потреба да додаваме градови! Системот е поврзан со OpenWeatherMap и работи за **секој град на светот**."

**Демонстрација**:
1.  Отвори **GET /api/weather/{city}**.
2.  Кликни **Try it out**.
3.  Во полето `city` напиши било што: `Skopje`, `Bitola`, `London`, `Miami`.
4.  Кликни **Execute**.
5.  Системот веднаш ќе врати температура и порака за достава.

---

## 4. "Што прави ова Weather API?"
**Одговор**: "Ако пробаме да нарачаме во град каде што врне (на пр. Лондон), системот ќе врати предупредување."
*   Можеш да им го покажеш `WeatherService.cs` кодот каде што е логиката: `if (condition.Contains("Rain"))`.

## 5. "Каде е базата?"
**Одговор**: "Користиме SQL база (SQLite за развој), а таа се креира и полни автоматски преку кодот во `Program.cs` и `DbSeeder`."
