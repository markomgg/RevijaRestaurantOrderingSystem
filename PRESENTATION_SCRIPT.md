# Детална Скрипта за Презентација: Revija Restaurant Ordering System

## 1. Вовед
"Здраво на сите. Денес ќе ви го презентирам проектот **Revija Restaurant Ordering System**.
Ова е комплетна backend апликација за менаџирање на нарачки во ресторан, изградена на **.NET 8** платформата, следејќи ја **Onion Architecture** структурата за максимална скалабилност и чист код."

---

## 2. Domain Слој (Ентитети)
"Ќе започнам од најважниот дел - срцето на апликацијата, а тоа е Domain слојот. Тука се дефинирани моделите кои го опишуваат нашиот бизнис."

*   **Customer (Клиент)**: "Оваа класа ги чува податоците за клиентот. Содржи `FullName`, `Email`, `Phone` и `Address`. Овие информации се клучни за доставата."
*   **MenuItem (Ставка од Мени)**: "Ова е моделот за храна и пијалоци. Секој производ има `Name` (на пр. 'Classic Burger'), `Description`, `Price`, `Category` (Бургери, Пијалоци) и `IsAvailable` знаменце што ни кажува дали производот е моментално достапен."
*   **Order (Нарачка)**: "Ова е главниот модел што ги поврзува работите. Една нарачка е врзана за еден `Customer`, има `OrderDate` (кога е направена) и пресметано `TotalPrice`."
*   **OrderItem (Ставка во Нарачка)**: "Бидејќи една нарачка може да има повеќе јадења, користиме `OrderItem`. Оваа класа кажува кој `MenuItem` е нарачан, во колкава количина (`Quantity`) и која била цената во тој момент."

---

## 3. Repository Слој (Инфраструктура)
"Податоците не лебдат во воздух, туку се чуваат во база. За тоа е задолжен Repository слојот."

*   **DbContext**: "Користиме Entity Framework Core. Имаме `RevijaDbContext` класа која служи како мост помеѓу нашиот C# код и базата на податоци. Таа ги мапира нашите ентитети во табели."
*   **Repositories**: "Имплементирав Pattern наречен Repository Pattern (преку класите `MenuItemRepository` и `OrderRepository`). Ова значи дека контролерите никогаш не пипуваат директно во база. Тие бараат од репозиториумот 'дај ми ги сите нарачки', а репозиториумот се грижи за SQL логиката. Ова ни овозможува лесно да ја смениме базата во иднина без да го менуваме целиот код."
*   **Data Seeding**: "Во овој слој додадов и `DbSeeder`, кој автоматски ја полни базата со почетни податоци (менито и тест корисници) штом апликацијата ќе се стартува."

---

## 4. Service Слој (Бизнис Логика)
"Service слојот е 'мозокот' на апликацијата. Тука се носат одлуките."

*   **OrderService**: "Кога ќе пристигне нарачка, овој сервис пресметува вкупна цена, проверува достапност и ги зачувува податоците."
*   **Weather Integration (Smart Delivery)**: "Најинтересниот дел е интеграцијата со надворешно API за време (OpenWeatherMap).
    Во `WeatherService`, пред да се потврди доставата, системот проверува каква е временската прогноза за тој град.
    Ако сервисот детектира дожд, снег или многу ниски температури, тој враќа 'Smart Alert' – предупредување до клиентот дека доставата може да доцни поради лизгави патишта. Ова додава вредност на апликацијата која оди подалеку од обичен CRUD."

---

## 5. Web API (Swagger)
"Како клиентите (React, Mobile App) комуницираат со сето ова? Преку Web API слојот."

*   **Controllers**: "Имаме контролери (`MenuItemsController`, `OrdersController`) кои изложуваат HTTP endpoints (GET, POST)."
*   **Swagger UI**: "За да го тестираме и визуелизираме ова, користиме Swagger. Тоа е оваа веб страна што ја гледате. Таа автоматски ја генерира документацијата.
    На пример, ако го отворам `/api/menu-items` и кликнам 'Execute', веднаш добивам JSON листа со сите бургерчиња од базата."

---

## 6. Хостирање и Cloud (Render)
"За крај, оваа апликација не работи само на мојот лаптоп. Таа е целосно хостирана на Cloud."

*   **Docker**: "Прво ја 'пакував' апликацијата во **Docker Container**. Ова гарантира дека таа ќе работи исто на секоја машина."
*   **Render.com**: "Потоа го поврзав мојот GitHub со платформата **Render**.
    Поставивме CI/CD (Continuous Deployment) – што значи дека секој пат кога ќе направам `git push`, Render автоматски го презема новиот код, го гради Docker контејнерот и ја пушта новата верзија во живо.
    Моментално апликацијата е достапна јавно на интернет преку HTTPS линк."
